This page demonstrates different ways to embed the booking system into your website.

Note: Replace your-booking-system.com with your actual booking system domain, and update the showId and performanceId with real values from your system.

1. Basic JavaScript Widget

The simplest way to embed a booking widget with responsive design.

<div id="booking-widget-1"></div> <script src="https://your-booking-system.com/booking-widget.js"></script> <script> createBookingWidget({ containerId: 'booking-widget-1', showId: 'abc123', performanceId: 'def456', responsive: true }); </script>

3. Auto-initialization with Data Attributes

Automatically initializes widgets using data attributes - no JavaScript required!

<div data-booking-widget data-show-id="abc123" data-performance-id="def456" data-responsive="true"> </div> <script src="https://your-booking-system.com/booking-widget.js"></script>

Auto-initialized booking widget would appear here

5. Direct Iframe Embedding

Simple iframe without JavaScript - works everywhere.

<iframe src="https://your-booking-system.com/embed/abc123/def456" width="800" height="600" frameborder="0" style="border: 1px solid #e2e8f0; border-radius: 8px; max-width: 100%;"> </iframe>

Direct iframe booking widget would appear here

Technical Information

Important: Before implementing, make sure to:
  • Replace placeholder URLs with your actual booking system domain
  • Use real show IDs and performance IDs from your system
  • Test the embedding on your actual website
  • Check mobile responsiveness

Getting Show and Performance IDs

To find the correct IDs for your shows:

  1. Go to your booking system admin dashboard
  2. Navigate to Shows â†’ View show details
  3. Copy the Show ID and Performance ID from the URL or details page
  4. Or use the API: GET /api/shows to list all shows and performances

Security Considerations

  • The embed system includes CORS headers for cross-origin requests
  • Iframe embedding is enabled with appropriate security headers
  • All booking data is transmitted securely over HTTPS
  • Payment processing (if implemented) follows PCI DSS guidelines

Browser Compatibility

  • Chrome 60+
  • Firefox 55+
  • Safari 12+
  • Edge 79+
  • Mobile browsers (iOS Safari, Chrome Mobile)
